## LinkedIn Post

Today in `#llmzoomcamp`, I pushed the Agentic RAG work a bit further.

This was more incremental than my first post, but technically it was an important step.

I expanded the helper notebook beyond a simple OpenAI smoke test into a fuller RAG and agent playground.

What changed:

- added `RAGBase`-style retrieval and prompt orchestration experiments
- pushed the notebook from one-off API calls toward a reusable RAG flow
- explored agent-loop patterns and function-calling style interactions
- kept comparing how retrieval quality shapes the final answer quality

The most useful technical lesson for me is that Agentic RAG is not just:

question -> LLM -> answer

It is really a system with multiple moving parts:

- retrieval
- context construction
- prompting
- tool use
- iteration loop

Once those pieces are explicit, it becomes much easier to debug why an answer is good, weak, or off-target.
